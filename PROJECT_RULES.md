0. Назначение документа

Этот документ — архитектурная конституция проекта.

Любой:

код

новая страница

изменение URL

изменение навигации

Если реализация противоречит документу — документ имеет приоритет.

1. Продуктовая цель

Проект — SEO-платформа image tools
(compress, resize, convert, crop и др.)

Цель:

масштабирование до десятков инструментов

масштабирование до тысяч SEO-страниц

единая архитектура

повторно используемый шаблон

бесплатная browser-only обработка

готовность к будущей server-side версии

1.1. Основной принцип

1 URL = 1 инструмент = 1 поисковый интент

Запрещено:

объединять инструменты на одной странице

делать “универсальные all-in-one” страницы

смешивать несколько интентов

2. SEO и URL-правила (обязательно)
2.1. Принцип

1 URL = 1 интент = 1 страница

2.2. Разрешённые URL-паттерны

Примеры:

/compress-image
/resize-image
/convert-image
/crop-image
/compress-jpg
/resize-jpg
/image-to-pdf
2.3. Запрещено

❌ Комбинировать интенты в URL
/compress-jpg-under-1mb-for-email

❌ Использовать синонимы
/reduce-image
/shrink-photo

❌ Использовать query-параметры для SEO-страниц

❌ Делать дубли страниц под один интент

2.4. Canonical

Каждая страница имеет canonical на себя

Синонимичные или устаревшие URL → 301 на основной

3. Архитектура страниц
3.1. Шаблон страницы (фиксированный порядок)

SEO Head (title, description, canonical, schema)

Header

Hero (H1 + subtitle)

Tool block (upload → process → download)

Results (до/после)

Ad slot (lazy)

SEO content

FAQ (+ FAQ schema)

Related tools

Footer

❗ Все инструменты используют один и тот же React/HTML шаблон.
Создание новых шаблонов запрещено.

4. Конфигурационная модель страниц
4.1. Страницы описываются конфигом

Страницы НЕ создаются копированием файлов.

Они описываются через конфигурацию (JSON/TS):

slug

intent

meta

параметры инструмента

related tools

Контент может генерироваться автоматически.

4.2. Intent (обязателен)

Каждая страница имеет ровно один intent:

base

format

size

platform

device

quality

batch

5. Product Architecture & Tools
5.1. Один инструмент = один интент

Каждый инструмент:

имеет отдельный URL

реализует одну задачу

не комбинируется с другими инструментами

❌ Запрещено:

compress + resize на одной странице

convert + compress

crop + resize + compress

5.2. Категории инструментов

Разрешённые категории:

Optimize
(compress, resize, crop, rotate, flip)

Convert
(image ↔ image, image ↔ pdf)

Format-specific
(compress-jpg, compress-png и т.д.)

Добавление новой категории требует согласования.

6. Навигация (обязательно)
6.1. Header navigation

Header используется только для основных entry-point инструментов.

Правила:

максимум 6–8 ссылок в dropdown

только core tools (base-intent)

формат-специфичные страницы запрещены в header

header — UX-инструмент, не SEO

6.2. Footer navigation

Footer содержит полный список инструментов.

Разрешено:

группировка по категориям

до 15 ссылок

plain links

Footer используется для SEO и навигации power users.

6.3. Related tools (обязательно)

Каждая страница инструмента обязана содержать блок Related tools.

Правила:

2–4 ссылки

только логически связанные инструменты

текущая страница не включается

анкор = название инструмента

Related tools — основной механизм внутренней перелинковки.

7. Compression — ключевые правила
7.1. Бесплатная версия (MVP)

Browser-only

Без серверного хранения

Без логирования файлов

Обработка через WebWorker

7.2. Форматы (MVP)

JPG / JPEG

PNG

WebP

7.3. Пресеты

Fast

Balanced (default)

Max

7.4. Ограничения (конфигурируемые)
maxFiles:
  single: 1
  batch: до 5

maxFileSize: 10 MB
maxTotalSize: 25 MB
soft pixel limit: ~25–30 MP
7.5. Target size

Используется только на отдельных страницах:

under 1MB

500KB

На остальных страницах — “уменьшить насколько возможно”.

7.6. Формат

Формат сохраняется (JPG → JPG, PNG → PNG).

8. Обязательная абстракция обработки

UI не знает, где происходит обработка.

process(
  file,
  options
) → {
  outputBlob,
  stats
}

Реализации:

browserProcess (текущая)

serverProcess (будущее)

UI не должен быть привязан к конкретной библиотеке.

9. Загрузка файлов

MVP:

локальный компьютер (drag & drop / browse)

Будущее:

Google Drive

Dropbox

Источники добавляются через адаптеры.

10. Хранение файлов

В бесплатной версии:

файлы не сохраняются

не кешируются

не логируются

не отправляются на сервер

11. Архитектурные требования

SEO-first (SSG или SSR)

Быстрый LCP

Минимальный CLS

Lazy loading тяжёлых частей

Sitemap генерируется автоматически из конфигов

12. Правила добавления новых инструментов

При разработке нового инструмента:

используется существующий шаблон страницы

не создаются новые layout или UI-паттерны

инструмент интегрируется в:

header (если core tool)

footer (всегда)

related tools

Любые отклонения запрещены без согласования.
